  <name>Task 2: Build CompanyInvitationForm with pricing configuration, batch mode, and email preview</name>
  <files>EventCenter.Web/Components/Pages/Admin/Events/CompanyInvitationForm.razor</files>
  <action>
    Create `CompanyInvitationForm.razor` as a dual-mode routed page.

    **Routes:**
    - `@page "/admin/events/{EventId:int}/companies/create"` (single create)
    - `@page "/admin/events/{EventId:int}/companies/create-batch"` (batch create)
    - `@page "/admin/events/{EventId:int}/companies/edit/{InvitationId:int}"` (edit existing)
    - `@attribute [Authorize(Policy = "AdminOnly")]`

    **Injected services:**
    - `@inject CompanyInvitationService CompanyInvitationService`
    - `@inject EventService EventService`
    - `@inject NavigationManager Navigation`

    **Mode detection in OnInitializedAsync:**
    - Determine mode from route: single-create, batch, or edit
    - Single-create: initialize empty CompanyInvitationFormModel, load event with agenda items, populate AgendaItemPrices from event's agenda items (BasePrice = CostForMakler)
    - Edit: load existing invitation, map to CompanyInvitationFormModel, populate AgendaItemPrices with current custom prices as ManualOverride
    - Batch: initialize list of CompanyInvitationFormModel (start with one empty row, add more via button)

    **SINGLE MODE FORM — wrapped in `<EditForm Model="@Model" OnValidSubmit="@HandleValidSubmit">`:**

    **Section 1: Firmendaten (card)**
    - CompanyName: `<InputText>` with label "Firmenname *"
    - ContactEmail: `<InputText type="email">` with label "Kontakt-E-Mail *"
    - ContactPhone: `<InputText>` with label "Telefon (optional)"
    - FluentValidationValidator for real-time validation
    - ValidationMessage for each field

    **Section 2: Preisgestaltung (card)** — per user decision
    - Percentage discount field at top:
      - Label: "Prozentrabatt (optional)"
      - `<InputNumber>` with "%" suffix in input-group
      - Help text: "Wird auf alle Agendapunkte angewendet"
      - On change: `@bind-Value:after="ApplyPercentageDiscount"` recalculates discounted prices

    - Per-item pricing table:
      - Header row: Agendapunkt | Basispreis | Rabattpreis | Individueller Preis
      - For each item in Model.AgendaItemPrices:
        - Column 1: AgendaItemTitle (text)
        - Column 2: BasePrice formatted as currency (read-only, `text-muted` — per user decision: "base price displayed as reference")
        - Column 3: Calculated discounted price (if percentage set, show in `text-success`; otherwise show dash)
        - Column 4: `<InputNumber>` for ManualOverride with "€" prefix, placeholder "Optional"
      - Use `table table-sm` for compact layout

    - `ApplyPercentageDiscount()` helper in @code:
      - When percentage changes, do NOT auto-populate ManualOverride fields (let admin explicitly override)
      - Only show calculated discount in "Rabattpreis" column as visual reference
      - This allows the percentage to show what the discount WOULD be, while ManualOverride column lets admin set a different final price

    **Section 3: Personliche Nachricht (card)**
    - `<InputTextArea>` rows=4, placeholder "Diese Nachricht wird in der Einladungs-E-Mail angezeigt..."
    - Label: "Personliche Nachricht (optional)"

    **Section 4: E-Mail-Vorschau (card, collapsible)** — per user decision
    - "Vorschau anzeigen" toggle button (btn-outline-info, bi-eye)
    - When expanded: render email preview HTML in an iframe or sanitized div
    - Call `CompanyInvitationService.BuildInvitationPreviewHtml(...)` to get preview content
    - If service doesn't have this method yet, build preview client-side using same template structure
    - Show warning if invitation hasn't been saved yet: "Speichern Sie zunachst die Einladung, um eine Vorschau zu sehen."

    **Action buttons (per user decision: "Save as draft" or "Create & Send"):**
    - "Als Entwurf speichern" (btn-secondary) — sets `Model.SendImmediately = false`, submits form
    - "Erstellen & Senden" (btn-primary) / "Speichern & Senden" in edit mode — sets `Model.SendImmediately = true`, submits form
    - "Abbrechen" (btn-outline-secondary) — navigates back to company list

    **In edit mode additional button:**
    - "Vorschau senden" (btn-info, bi-envelope-open) — sends email preview to admin's own email (stretch goal, skip if complex)

    **HandleValidSubmit logic:**
    - Single-create: `CompanyInvitationService.CreateInvitationAsync(EventId, Model)`
    - Edit: `CompanyInvitationService.UpdateInvitationAsync(InvitationId, Model)`, then if SendImmediately, also call SendInvitationAsync
    - On success: navigate to company list with success message
    - On failure: show error alert at top of form

    **BATCH MODE FORM:**
    - Title: "Sammeleinladung — Standardpreise"
    - Help text: "Alle eingeladenen Firmen erhalten die Standard-Veranstaltungspreise."
    - Dynamic list of company rows (initially 1):
      - Each row: CompanyName input + ContactEmail input + remove button (bi-x-circle)
      - "Weitere Firma hinzufugen" button (btn-outline-primary, bi-plus)
    - Optional shared percentage discount (applies to all)
    - Optional shared personal message
    - Submit: "Einladungen erstellen & versenden" (btn-primary)
    - Calls `CompanyInvitationService.CreateBatchInvitationsAsync(...)`
    - Shows results: "{succeeded} Einladungen erstellt, {failed} fehlgeschlagen" with error details

    **Pricing warning for Booked status (edit mode):**
    - If editing an invitation with Status == Booked, show Bootstrap alert (warning):
    - "Achtung: Diese Firma hat bereits gebucht. Preisanderungen betreffen die zukunftige Abrechnung."

    **German text** throughout with proper Umlauts.
