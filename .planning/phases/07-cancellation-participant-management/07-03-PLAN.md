**Step 5: Add modal helper methods.**
In the `@code` block:

```csharp
private void OpenCancelModal(int registrationId, string name, bool isOwnRegistration)
{
    cancellingRegistrationId = registrationId;
    cancellingRegistrationName = isOwnRegistration ? "Ihre Anmeldung" : name;
    cancellationReason = null;
    cancelSuccessMessage = null;
    cancelErrorMessage = null;
    showCancelModal = true;
}

private void CloseCancelModal()
{
    showCancelModal = false;
    cancellingRegistrationId = null;
}

private async Task ConfirmCancellation()
{
    if (cancellingRegistrationId == null) return;

    isCancelling = true;
    try
    {
        var result = await RegistrationService.CancelRegistrationAsync(
            cancellingRegistrationId.Value,
            userEmail,
            cancellationReason);

        if (result.Success)
        {
            showCancelModal = false;
            cancelSuccessMessage = "Anmeldung erfolgreich storniert.";
            cancelErrorMessage = null;

            // Refresh page state: re-check registration status
            await LoadPageData();
        }
        else
        {
            cancelErrorMessage = result.ErrorMessage;
            showCancelModal = false;
        }
    }
    finally
    {
        isCancelling = false;
    }
}
```

**Step 6: Add a LoadPageData helper method** (or identify the existing data loading in OnInitializedAsync and extract into a reusable method) that:

- Reloads guest data
This enables the page to reflect the cancellation immediately.

Important: When checking `isUserRegistered`, the existing code likely uses `evt.Registrations.FirstOrDefault(...)`. After cancellation, this check must filter `!r.IsCancelled` so a cancelled registration is no longer treated as "registered." Verify and fix the existing registration check:
