**Step 1: Implement CancelRegistrationAsync in RegistrationService.**
Add this method to `EventCenter.Web/Services/RegistrationService.cs`:

```csharp
public async Task<(bool Success, string? ErrorMessage)> CancelRegistrationAsync(
    int registrationId,
    string cancelledByEmail,
    string? cancellationReason)
```

Implementation logic (follow research pattern from 07-RESEARCH.md):

1. Load registration with `.Include(r => r.Event).Include(r => r.ParentRegistration)`.
2. If registration is null → return `(false, "Anmeldung nicht gefunden.")`
3. If registration.IsCancelled → return `(false, "Anmeldung ist bereits storniert.")`
4. Check event deadline using `registration.Event.GetCurrentState()`. If NOT `EventState.Public` → return `(false, "Stornierung nach Anmeldefrist nicht möglich.")`
5. Permission check:
   - `isOwner`: `registration.Email.Equals(cancelledByEmail, StringComparison.OrdinalIgnoreCase)`
   - `isGuestOwner`: `registration.ParentRegistration?.Email.Equals(cancelledByEmail, StringComparison.OrdinalIgnoreCase) ?? false`
   - If neither → return `(false, "Keine Berechtigung zum Stornieren dieser Anmeldung.")`
6. Set `registration.IsCancelled = true`, `registration.CancellationDateUtc = DateTime.UtcNow`, `registration.CancellationReason = cancellationReason`
7. Call `await _context.SaveChangesAsync()`
8. **CRITICAL per user decision:** Do NOT cancel guest registrations when broker cancels. Only the specific registration is cancelled.
9. Fire-and-forget email sending (established pattern):
```csharp
_ = Task.Run(async () =>
{
    try
    {
        await _emailSender.SendMaklerCancellationConfirmationAsync(registration);
        await _emailSender.SendAdminMaklerCancellationNotificationAsync(registration);
    }
    catch (Exception ex)
    {
        _logger.LogError(ex, "Failed to send cancellation emails for registration {RegistrationId}", registrationId);
    }
});
```
10. Return `(true, null)`.
