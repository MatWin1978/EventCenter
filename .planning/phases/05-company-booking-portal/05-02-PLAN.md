<feature>
  <name>CompanyBookingService</name>
  <files>EventCenter.Web/Services/CompanyBookingService.cs, EventCenter.Tests/Services/CompanyBookingServiceTests.cs</files>
  <behavior>
    The service manages the entire company booking lifecycle:

    **ValidateInvitationCodeAsync(string invitationCode)**
    - Input: invitation code string from URL
    - Returns: `(bool IsValid, EventCompany? Company, string? ErrorMessage)`
    - Uses CryptographicOperations.FixedTimeEquals for constant-time comparison (prevent timing attacks)
    - Loads invitation codes from database, compares in-memory with constant-time comparison
    - Checks expiration via ExpiresAtUtc field (if set)
    - Checks that invitation status is Sent (not Draft, not already Booked unless returning to view status)
    - Includes Event, AgendaItems, AgendaItemPrices, EventOptions navigation properties
    - Invalid/expired → returns friendly German error messages
    - Cases:
      - Valid code, not expired, status=Sent → (true, company, null)
      - Valid code, status=Booked → (true, company, null) — returning visitor sees status page
      - Valid code, status=Cancelled → (true, company, null) — shows cancelled status with re-booking option
      - Invalid code → (false, null, "Dieser Link ist ungültig oder abgelaufen.")
      - Expired code → (false, null, "Dieser Link ist abgelaufen. Bitte kontaktieren Sie uns für eine neue Einladung.")
      - Status=Draft → (false, null, "Diese Einladung wurde noch nicht versendet.")

    **SubmitBookingAsync(int eventCompanyId, CompanyBookingFormModel formModel)**
    - Input: invitation ID and booking form
    - Returns: `(bool Success, string? ErrorMessage)`
    - Uses Database.BeginTransactionAsync for atomicity
    - Updates EventCompany: Status → Booked, BookingDateUtc → DateTime.UtcNow
    - Creates Registration entity for EACH participant:
      - RegistrationType = CompanyParticipant
      - EventCompanyId set to invitation ID
      - FirstName, LastName, Email from ParticipantModel
      - RegistrationDateUtc = DateTime.UtcNow
      - IsConfirmed = true
    - Links RegistrationAgendaItem entries for each participant's selected agenda items
    - Links selected extra options to first registration (or separate handling)
    - Fire-and-forget admin notification via SendAdminBookingNotificationAsync (MAIL-04)
    - Commits transaction, returns success
    - Error: invitation not found → returns error
    - Error: invitation not in Sent status → "Diese Einladung kann nicht mehr gebucht werden."
    - Cases:
      - Valid submission with 2 participants → creates 2 Registrations + agenda items, status=Booked
      - Invalid invitation ID → error
      - Already booked → error

    **CancelBookingAsync(int eventCompanyId, string? cancellationComment)**
    - Input: invitation ID and optional comment
    - Returns: `(bool Success, string? ErrorMessage)`
    - Updates EventCompany: Status → Cancelled, CancellationComment, IsNonParticipation = false
    - Marks all linked Registrations as IsCancelled = true, CancellationDateUtc = DateTime.UtcNow
    - Fire-and-forget admin notification via SendAdminCancellationNotificationAsync (MAIL-05)
    - Only works when status = Booked
    - Cases:
      - Booked invitation → cancels, returns success
      - Non-booked → error "Keine aktive Buchung vorhanden."

    **ReportNonParticipationAsync(int eventCompanyId, string? comment)**
    - Input: invitation ID and optional comment
    - Returns: `(bool Success, string? ErrorMessage)`
    - Updates EventCompany: Status → Cancelled, CancellationComment = comment, IsNonParticipation = true
    - Marks all linked Registrations as IsCancelled = true
    - Fire-and-forget admin notification with isNonParticipation = true (MAIL-05)
    - Only works when status = Booked
    - Cases:
      - Booked invitation → marks non-participation, returns success
      - Non-booked → error

    **GetBookingStatusAsync(int eventCompanyId)**
    - Input: invitation ID
    - Returns: `EventCompany?` with Registrations and Event loaded
    - Used by UI to show booking status on return visits

    **CalculateTotalCost(EventCompany company, CompanyBookingFormModel formModel, List<EventAgendaItem> agendaItems, List<EventOption> eventOptions)**
    - Pure calculation method (no DB access)
    - For each participant: sum prices of selected agenda items (use company-specific CustomPrice from AgendaItemPrices, fallback to CostForMakler)
    - Add prices of selected extra options
    - Returns decimal total
    - Cases:
      - 2 participants, each with 2 agenda items at custom prices → correct total
      - With extra options → adds option prices to total
      - No custom price (null CustomPrice) → uses base CostForMakler
  </behavior>
