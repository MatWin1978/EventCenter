  <action>
    Create directory `EventCenter.Web/Components/Pages/Company/` if it doesn't exist.

    Create `CompanyBooking.razor` — a single Blazor page handling the entire company booking lifecycle. The page routes at `/company/booking/{InvitationCode}` with `@attribute [AllowAnonymous]`.

    **CRITICAL: Anonymous access configuration.**
    The app uses `AuthorizeRouteView` in Routes.razor which requires auth by default. The `[AllowAnonymous]` attribute on the page should override this. If this doesn't work (WebSocket disconnect issue documented in research), the executor may need to add `[AllowAnonymous]` handling in Routes.razor or add a fallback. Test by navigating to the page without authentication.

    **Page structure (single scrollable page per user decision):**

    The page has multiple view states managed by an enum or flags:
    - `Loading` — spinner while validating GUID
    - `Error` — friendly error page for invalid/expired GUID
    - `BookingForm` — the main booking form (status = Sent)
    - `BookingSubmitted` — simple success confirmation
    - `BookingStatus` — status view for return visitors (status = Booked)
    - `CancelledStatus` — shows cancelled/non-participation status with optional re-booking

    **1. OnInitializedAsync:**
    - Call `CompanyBookingService.ValidateInvitationCodeAsync(InvitationCode)`
    - If invalid → show Error view with message: "Dieser Link ist ungültig oder abgelaufen. Bitte kontaktieren Sie uns unter [evt.ContactEmail or admin@example.com] für eine neue Einladung."
    - If valid and Status = Sent → show BookingForm
    - If valid and Status = Booked → load booking status, show BookingStatus view
    - If valid and Status = Cancelled → show CancelledStatus with re-booking option if deadline not passed

    **2. Error view:**
    - Container with alert-danger card
    - Icon (bi-exclamation-triangle-fill)
    - Message: from service or default "Dieser Link ist ungültig oder abgelaufen."
    - Contact info: "Bitte kontaktieren Sie uns unter {contactEmail} für eine neue Einladung."
    - No navigation links (anonymous user has no other pages to visit)

    **3. BookingForm view (main content):**

    **3a. Event summary header (compact per user decision):**
    - Card at top with: event title (h3), date (formatted CET), location, company name (badge)
    - No full event description — keep compact

    **3b. Participant table (inline editable rows):**
    - Card with header "Teilnehmer"
    - Responsive table with columns: Anrede, Vorname, Nachname, E-Mail, Agendapunkte, (delete button)
    - Each row is bound to a ParticipantModel in the form's Participants list
    - Anrede: InputSelect with options "Herr", "Frau", "Divers"
    - Vorname/Nachname: InputText with form-control class
    - E-Mail: InputText type="email"
    - Agendapunkte: checkboxes for each available agenda item (from EventCompany.Event.AgendaItems where MaklerCanParticipate = true), showing title and price. Use company-specific price if available (from AgendaItemPrices), otherwise fallback to CostForMakler.
    - Delete button: btn-sm btn-outline-danger with bi-trash icon (disabled if only one participant)
    - "Teilnehmer hinzufügen" button below table (btn-outline-primary, bi-plus-circle icon)
    - Table starts with one empty pre-filled row (per user decision)
    - Use `@for` loop with captured index for lambda closures
    - Call `StateHasChanged()` after add/remove/toggle operations to update cost summary

    **3c. Extra options section (if EventOptions exist for the event):**
    - Card with header "Zusatzoptionen"
    - Checkbox list of available EventOptions showing name, description, and price
    - Bind to `model.SelectedExtraOptionIds`
    - Call `StateHasChanged()` on toggle

    **3d. Sticky cost summary (sidebar on desktop, bottom bar on mobile):**
    - Use two-column Bootstrap layout on desktop (col-lg-8 for form, col-lg-4 for cost summary)
    - Cost summary card with `position: sticky; top: 20px;` CSS
    - Card header: "Kostenübersicht" (bg-success text-white)
    - Content sections:
      - "Teilnehmerkosten:" — per-participant breakdown showing participant name and cost
      - If extra options selected: separator + "Zusatzoptionen:" with option names and prices
      - Separator + "Gesamtkosten:" displayed prominently (h4, text-success)
      - "Alle Preise zzgl. MwSt." note (small text-muted)
    - TotalCost computed property that sums participant costs + option costs
    - GetAgendaItemPrice helper: check EventCompanyAgendaItemPrice.CustomPrice first, fallback to EventAgendaItem.CostForMakler
    - CalculateParticipantCost helper: sum prices of selected agenda items for a participant
    - Cost updates live as participants and options change (Blazor reactive binding + explicit StateHasChanged)
    - Use German culture formatting for currency: `price.ToString("C", new System.Globalization.CultureInfo("de-DE"))`

    **3e. Submit section:**
    - Wrap form in `<EditForm Model="model">` with `<FluentValidationValidator />`
    - `<ValidationSummary />` for overall form errors
    - Submit button: "Buchung absenden" (btn-success btn-lg)
    - Loading spinner on submit button while submitting (disable button, show spinner)
    - On submit: call `CompanyBookingService.SubmitBookingAsync(eventCompany.Id, model)`
    - If success → switch to BookingSubmitted view
    - If error → show alert-danger with error message

    **4. BookingSubmitted view (simple success per user decision):**
    - Container with card
    - Success icon (bi-check-circle-fill, text-success, large)
    - Heading: "Buchung erfolgreich eingereicht"
    - Message: "Sie erhalten in Kürze eine Bestätigung per E-Mail."
    - Company name and event title displayed for reference
    - No detailed summary on confirmation page (per user decision)

    **5. BookingStatus view (return visitors, status = Booked):**
    - Container with event summary header (same compact format)
    - Status badge: "Gebucht" (badge bg-success)
    - Booking date displayed
    - Participant list (read-only table showing registered participants)
    - Single action button: "Buchung bearbeiten" (btn-outline-primary)
    - On click: open Bootstrap modal with two choices:
      - "Buchung stornieren" (btn-danger) — cancels entire booking
      - "Nicht-Teilnahme melden" (btn-warning) — reports non-participation
    - Modal includes optional textarea for comment/reason
    - On cancel/non-participation:
      - Call appropriate service method (CancelBookingAsync or ReportNonParticipationAsync)
      - If success → switch to CancelledStatus view
      - If error → show alert in modal

    **6. CancelledStatus view (status = Cancelled):**
    - Container with event summary header
    - Status badge: "Storniert" (badge bg-danger) or "Nicht-Teilnahme" (badge bg-warning) based on IsNonParticipation
    - Cancellation date displayed
    - Cancellation comment displayed (if any)
    - If registration deadline has NOT passed:
      - Show "Erneut buchen" button (btn-primary)
      - On click: reset form and switch to BookingForm view
      - Service should handle re-booking by creating new registrations (EventCompany status transitions back to Booked)
    - If deadline passed: show message "Die Anmeldefrist ist abgelaufen."

    **Code-behind approach:**
    Use `@code { }` block at bottom of .razor file (following project pattern from EventRegistration.razor).

    **Key code-behind elements:**
    ```csharp
    [Parameter] public string InvitationCode { get; set; } = string.Empty;

    private EventCompany? eventCompany;
    private CompanyBookingFormModel model = new();
    private List<EventAgendaItem> availableAgendaItems = new();
    private List<EventOption> availableOptions = new();
    private bool isLoading = true;
    private bool isSubmitting = false;
    private string? errorMessage;
    private string? viewState; // "loading", "error", "form", "submitted", "booked", "cancelled"
    private bool showManagementModal = false;
    private string? managementComment;
    ```

    **Styling:**
    - Use Bootstrap 5 classes throughout (consistent with rest of app)
    - Bootstrap Icons for visual elements
    - German labels and text throughout (proper Umlauts)
    - Responsive: form takes full width on mobile, two-column (form + cost sidebar) on desktop
    - Add `<style>` block for sticky cost summary positioning and responsive overrides
    - Form validation: use Blazored.FluentValidation for inline field validation
