    **Create EventList.razor** at `Components/Pages/Admin/Events/EventList.razor`:

    Route: `@page "/admin/events"`
    Authorization: `@attribute [Authorize(Roles = "Admin")]`
    Layout: `@layout MainLayout`

    **Inject:** `@inject EventService EventService`

    **Page structure:**

    1. **Header section:**
       - `<h2>Veranstaltungen</h2>`
       - "Neue Veranstaltung" button linking to `/admin/events/create` (will be created in Plan 04)
       - Toggle checkbox/switch: "Vergangene anzeigen" to toggle `showPast` state

    2. **Data table:**
       - Use Bootstrap `<table class="table table-striped table-hover">` with `<thead>` and `<tbody>`
       - **Columns** (per user decision):
         - **Titel** — sortable (click header)
         - **Datum** — display StartDateUtc formatted as CET using `TimeZoneHelper.FormatDateTimeCet()`, sortable
         - **Ort** — sortable
         - **Status** — render `<EventStatusBadge State="@evt.GetCurrentState()" />`
         - **Anmeldungen** — display `{GetCurrentRegistrationCount()}/{MaxCapacity}`
         - **Veröffentlicht** — Bootstrap check icon if IsPublished
         - **Aktionen** — action buttons (see below)

       - **Sortable headers:** Each sortable column header is a clickable `<a>` or `<button>` that sets `currentSortColumn` and toggles `ascending`. Show sort direction indicator (arrow up/down) on active column.

    3. **Action buttons per row** (using Bootstrap button group or dropdown):
       - **Bearbeiten** — NavigationManager.NavigateTo(`/admin/events/edit/{evt.Id}`)
       - **Veröffentlichen** / **Zurückziehen** — toggles based on IsPublished. Opens ConfirmDialog before executing.
         - Publish message: "Diese Veranstaltung wird für alle Makler sichtbar. Fortfahren?"
         - Unpublish message: "Diese Veranstaltung wird für Makler nicht mehr sichtbar. Fortfahren?"
       - **Duplizieren** — calls EventService.DuplicateEventAsync(), then navigates to edit page for the duplicate
       - **Löschen** — opens ConfirmDialog with "Soll diese Veranstaltung wirklich gelöscht werden?" Only show if no registrations (or handle error from service).

    4. **Pagination:**
       - Show page numbers below table
       - Page size: 15 (Claude's discretion per user decision)
       - Show total count: "Zeige {from}-{to} von {total} Veranstaltungen"

    5. **Loading state:**
       - Show spinner or "Lade Veranstaltungen..." while data loads
       - Show "Keine Veranstaltungen gefunden." when empty

    **Code-behind (@code block):**
    - `List<Event> events` — current page data
    - `int totalCount`, `int currentPage = 1`, `int pageSize = 15`
    - `bool showPast = false` — toggle for past events
    - `string currentSortColumn = "StartDateUtc"`, `bool ascending = false`
    - `bool isLoading = true`
    - ConfirmDialog state: `bool showConfirmDialog`, `string confirmTitle`, `string confirmMessage`, `EventCallback confirmAction`

    - `OnInitializedAsync()` — call LoadEvents()
    - `LoadEvents()` — call EventService.GetEventsAsync() and GetEventCountAsync()
    - `SortBy(string column)` — toggle sort direction or change column, reload
    - `ChangePage(int page)` — update currentPage, reload
    - `TogglePast()` — flip showPast, reset page to 1, reload
    - `PublishEvent(int id)` / `UnpublishEvent(int id)` — show ConfirmDialog, on confirm call service
    - `DuplicateEvent(int id)` — call service, navigate to edit page
    - `DeleteEvent(int id)` — show ConfirmDialog, on confirm call service, handle error if registrations exist

    **Error handling:** Show alert message when unpublish or delete fails due to registrations. Use a `string? errorMessage` field displayed as Bootstrap alert.

    **Update Admin/Index.razor** — Add a link/card to "Veranstaltungen verwalten" pointing to `/admin/events`. Or update NavMenu to include Events link under Admin section.
