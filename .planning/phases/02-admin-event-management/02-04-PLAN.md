    **Routes (two routes, one component):**
    - `@page "/admin/events/create"` — create mode
    - `@page "/admin/events/edit/{EventId:int}"` — edit mode

    Authorization: `@attribute [Authorize(Roles = "Admin")]`
    Layout: `@layout MainLayout`

    **Inject:** `@inject EventService EventService`, `@inject NavigationManager NavigationManager`

    **Important CET date handling pattern:**
    The form must display dates in CET for the admin but store them as UTC. Create local DateTime fields for the CET representation:

    ```csharp
    // In @code block — CET display fields (bound to form inputs)
    private DateTime startDateCet;
    private DateTime endDateCet;
    private DateTime registrationDeadlineCet;

    // On form load (edit mode): convert UTC -> CET for display
    startDateCet = TimeZoneHelper.ConvertUtcToCet(
        DateTime.SpecifyKind(Model.StartDateUtc, DateTimeKind.Utc));

    // On form submit: convert CET -> UTC for storage
    Model.StartDateUtc = TimeZoneHelper.ConvertCetToUtc(startDateCet);
    ```

    **Form structure using EditForm + FluentValidationValidator:**

    ```razor
    <EditForm Model="@Model" OnValidSubmit="@HandleValidSubmit">
        <FluentValidationValidator />
        <ValidationSummary />

        <!-- Sections below -->
    </EditForm>
    ```

    **Section 1: Grunddaten (Basic Info)**
    - Titel (InputText, required) — `@bind-Value="Model.Title"`
    - Beschreibung (InputTextArea) — `@bind-Value="Model.Description"`
    - Ort (InputText, required) — `@bind-Value="Model.Location"`
    - Maximale Kapazität (InputNumber) — `@bind-Value="Model.MaxCapacity"`
    - Maximale Begleitpersonen (InputNumber) — `@bind-Value="Model.MaxCompanions"`

    **Section 2: Termine (Dates) — label "(CET)" per user decision**
    - Startdatum (CET) (InputDate type="datetime-local") — `@bind-Value="startDateCet"`
    - Enddatum (CET) (InputDate type="datetime-local") — `@bind-Value="endDateCet"`
    - Anmeldefrist (CET) (InputDate type="datetime-local") — `@bind-Value="registrationDeadlineCet"`

    Use `<input type="datetime-local" @bind="startDateCet" class="form-control" />` for datetime-local picker (Blazor InputDate doesn't support datetime-local well; use raw input element with @bind).

    **Section 3: Kontaktperson (Contact Person) — per user decision**
    - Name (InputText) — `@bind-Value="Model.ContactName"`
    - E-Mail (InputText) — `@bind-Value="Model.ContactEmail"`
    - Telefon (InputText) — `@bind-Value="Model.ContactPhone"`

    **Section 4: Dokumente (Documents)**
    - Display list of current documents with filename and "Entfernen" button
    - `<InputFile OnChange="@HandleFileUpload" multiple accept=".pdf,.jpg,.png,.jpeg" />`
    - Max file size: 10 MB (const `MaxFileSize = 10 * 1024 * 1024`)
    - On file selection: validate size, call `EventService.SaveDocumentAsync()` for each file (only works in edit mode when Id exists). In create mode, queue files and upload after initial save.
    - Show error for oversized files: "Datei {name} überschreitet die maximale Größe von 10 MB"
    - "Entfernen" button calls `EventService.DeleteDocumentAsync()` and removes path from Model.DocumentPaths

    **Form buttons at bottom:**
    - "Speichern" submit button (type="submit")
    - "Abbrechen" link back to `/admin/events`

    **OnInitializedAsync():**
    - If `EventId` parameter exists (edit mode): Load event via `EventService.GetEventByIdAsync(EventId)`. Convert UTC dates to CET for display fields. Set `isEditMode = true`.
    - If no `EventId` (create mode): Initialize `Model = new Event()` with sensible defaults (StartDateUtc = tomorrow, EndDateUtc = tomorrow + 1 day, RegistrationDeadlineUtc = tomorrow - 1 day, MaxCapacity = 50, MaxCompanions = 1). Convert to CET for display.

    **HandleValidSubmit():**
    1. Convert CET dates back to UTC:
       ```csharp
       Model.StartDateUtc = TimeZoneHelper.ConvertCetToUtc(startDateCet);
       Model.EndDateUtc = TimeZoneHelper.ConvertCetToUtc(endDateCet);
       Model.RegistrationDeadlineUtc = TimeZoneHelper.ConvertCetToUtc(registrationDeadlineCet);
       ```
    2. If create mode: call `EventService.CreateEventAsync(Model)`, then upload queued files, then navigate to `/admin/events`.
    3. If edit mode: call `EventService.UpdateEventAsync(Model)`, navigate to `/admin/events`.
    4. Show success message or error alert.

    **Error handling:**
    - Display validation errors using `<ValidationMessage>` per field or `<ValidationSummary />`
    - Show alert for save errors

    **Page title:**
    - Create: `<h2>Neue Veranstaltung anlegen</h2>`
    - Edit: `<h2>Veranstaltung bearbeiten</h2>`

    **CET handling for agenda item dates:**
    Since agenda items can have many items, use a Dictionary to track CET display values:
    ```csharp
    private Dictionary<EventAgendaItem, (DateTime StartCet, DateTime EndCet)> agendaItemDatesCet = new();
    ```
    Or use a simpler approach: convert all dates on form load and form submit, storing CET values directly in the entity's UTC fields during editing, then converting all at submit time. Choose whichever approach is cleaner. Document the pattern chosen.

    **AddAgendaItem():**
    ```csharp
    private void AddAgendaItem()
    {
        Model.AgendaItems.Add(new EventAgendaItem
        {
            MaklerCanParticipate = true,
            GuestsCanParticipate = true,
            StartDateTimeUtc = Model.StartDateUtc, // Default to event start
            EndDateTimeUtc = Model.EndDateUtc       // Default to event end
        });
    }
    ```

    **RemoveAgendaItem(EventAgendaItem item):**
    - Remove from Model.AgendaItems collection
    - If item has Id > 0 (existing in DB), mark for deletion (will be handled by EF Core change tracking when parent is saved, or explicitly delete via context)

    **Section 6: Zusatzoptionen (Extra Options) — inline per user decision**

    Same pattern as agenda items but simpler fields:
    - Name (InputText, required)
    - Beschreibung (InputTextArea, optional)
    - Preis (InputNumber, step="0.01") with "€" suffix
    - Max. Anzahl (InputNumber, optional)
    - "Entfernen" button — calls EventService.DeleteEventOptionAsync() if existing (Id > 0). Shows error if booked: "Diese Zusatzoption kann nicht gelöscht werden, da bereits Buchungen existieren." If new (Id == 0), simply remove from collection.

    **AddExtraOption():**
    ```csharp
    private void AddExtraOption()
    {
        var options = Model.EventOptions as ICollection<EventOption> ?? new List<EventOption>();
        options.Add(new EventOption());
        Model.EventOptions = options;
    }
    ```

    **Update HandleValidSubmit():**
    - Before saving, convert all agenda item dates from CET to UTC
    - For create mode: save event first to get Id, then save agenda items and options as children
    - For edit mode: the EF Core context tracks changes to child collections. Ensure AgendaItems and EventOptions are included when loading the event and modified through the same context.

    **Important EF Core consideration:**
    When editing, the form modifies the Event entity that was loaded from the context. AgendaItems and EventOptions modifications (add/remove/edit) should be tracked by EF Core. New items (Id == 0) will be added on SaveChanges. Modified items will be updated. Removed items need explicit handling — either use `context.AgendaItems.Remove()` for deleted items or manage a separate "removed items" list.

    Consider creating a method in EventService: `async Task SaveEventWithChildrenAsync(Event evt, List<int> removedAgendaItemIds, List<int> removedOptionIds)` that handles the full save including child collection management.
