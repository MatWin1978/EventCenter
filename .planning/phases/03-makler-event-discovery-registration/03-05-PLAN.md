       - Use Bootstrap form-group layout
       - Email from auth state should be readonly (makler registers with their Keycloak email)

    3. **Agenda item selection section (Claude's discretion on presentation style — use checklist):**
       - `<h4>Agendapunkte auswählen</h4>`
       - For each agenda item where MaklerCanParticipate:

           ```html
           <div class="form-check card p-3 mb-2">
             <input type="checkbox" class="form-check-input"
                    checked="@IsItemSelected(item.Id)"
                    @onchange="() => ToggleAgendaItem(item.Id)" />
             <label class="form-check-label">
               <strong>@item.Title</strong>
               @if (item.IsMandatory) { <span class="badge bg-warning text-dark ms-2">Pflicht</span> }
               <br/>
               <small class="text-muted">
                 @FormatDateTimeCet(item.StartDateTimeUtc, "dd.MM.yyyy HH:mm") - @FormatDateTimeCet(item.EndDateTimeUtc, "HH:mm")
               </small>
               <br/>
               <span>@(item.CostForMakler > 0 ? $"{item.CostForMakler:N2} EUR" : "Kostenfrei")</span>
             </label>
           </div>
           ```

         - Validation message if no items selected: "Bitte wählen Sie mindestens einen Agendapunkt aus."

       - `<h4>Kostenübersicht</h4>`
       - Table or list showing each selected item with its cost
       - **Total row:** Bold total in EUR
       - Reactively updates as checkboxes change

    5. **Submit button:**
       - `<button type="button" class="btn btn-primary btn-lg" @onclick="ShowConfirmation">Anmeldung abschließen</button>`
       - Disabled if form invalid or no items selected
       - Back link: `<a href="/portal/events/@EventId" class="btn btn-outline-secondary ms-2">Abbrechen</a>`

    6. **Confirmation modal (per locked decision: modal before final submission):**
       - Use ConfirmDialog component (from Phase 02 shared components)
       - Title: "Anmeldung bestätigen"
       - Message: Summary of selected agenda items and total cost
       - Build message dynamically:
         ```
         Sie möchten sich für folgende Agendapunkte anmelden:
         - Item 1: X,XX EUR
         - Item 2: Y,YY EUR
         Gesamtkosten: Z,ZZ EUR
         ```
       - ConfirmText: "Anmeldung bestätigen"
       - On confirm: call `SubmitRegistration()`

    **SubmitRegistration logic:**
    ```csharp
    private async Task SubmitRegistration()
    {
        isSubmitting = true;
        showConfirmDialog = false;

        var (success, registrationId, error) = await RegistrationService.RegisterMaklerAsync(
            EventId, model.Email, model.FirstName, model.LastName,
            model.Phone, model.Company, model.SelectedAgendaItemIds);

        if (success && registrationId.HasValue)
        {
            NavigationManager.NavigateTo($"/portal/registrations/{registrationId.Value}/confirmation");
        }
        else
        {
            errorMessage = error ?? "Ein Fehler ist aufgetreten.";
            isSubmitting = false;
        }
    }
    ```

    **Error handling:**
    - Display error message from RegistrationService in alert-danger
    - Dismissible error alert
    - "isSubmitting" flag prevents double-submit (disable button + show spinner)

    **Code organization:**
    - RegistrationFormModel bound to EditForm
    - SelectedAgendaItemIds managed via checkbox toggle methods
    - Cost calculated from selected items: `selectedItems.Sum(i => i.CostForMakler)`
    - Mandatory items auto-selected on init
