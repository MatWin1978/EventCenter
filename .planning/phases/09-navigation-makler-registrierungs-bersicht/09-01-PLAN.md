    Replace the entire content of Home.razor with a redirect-only component. The component renders nothing during the async check and shows only the "Zugriff verweigert" message if authenticated with no known role.

    Logic (in OnInitializedAsync):
    1. Inject NavigationManager, AuthenticationStateProvider
    2. Get auth state
    3. If not authenticated → NavigateTo("/auth/login", forceLoad: false)
    4. If IsInRole("Admin") → NavigateTo("/admin/events", forceLoad: false)
    5. If IsInRole("Makler") → NavigateTo("/portal/events", forceLoad: false)
    6. Otherwise → set `showAccessDenied = true` (no auto-logout)

    Template: Keep empty except for the conditional "Zugriff verweigert" alert. Use `@if (showAccessDenied)` guard so nothing renders during the async check — prevents content flash.

    Required directive: `@page "/"` (keep existing page route)
    Required usings: `@using Microsoft.AspNetCore.Components.Authorization`
    Inject: `@inject NavigationManager Navigation` and `@inject AuthenticationStateProvider AuthStateProvider`

    Access denied message (German):
    ```html
    <div class="alert alert-warning mt-4">
        <h4>Zugriff verweigert</h4>
        <p>Ihr Konto hat keine bekannte Rolle. Bitte wenden Sie sich an Ihren Administrator.</p>
    </div>
    ```

    Note: Do NOT use forceLoad: true — internal Blazor navigation uses forceLoad: false. Do NOT add [Authorize] attribute — the page must be accessible to unauthenticated users so they can be redirected to /auth/login. The existing Routes.razor AuthorizeRouteView with RedirectToLogin also handles unauthenticated users, but the explicit check in OnInitializedAsync is the belt-and-suspenders approach from RESEARCH.md.

  <action>
    Add a new public method `GetBrokerRegistrationsAsync(string brokerEmail)` at the end of the RegistrationService class (before the closing brace), returning `Task<List<Registration>>`.

    The method must use the following Include chain to avoid N+1 queries (lazy loading is not configured in this project):
    ```csharp
    public async Task<List<Registration>> GetBrokerRegistrationsAsync(string brokerEmail)
    {
        return await _context.Registrations
            .Include(r => r.Event)
            .Include(r => r.RegistrationAgendaItems)
                .ThenInclude(rai => rai.AgendaItem)
            .Include(r => r.SelectedOptions)
            .Include(r => r.GuestRegistrations)
                .ThenInclude(g => g.RegistrationAgendaItems)
                    .ThenInclude(rai => rai.AgendaItem)
            .Where(r =>
                r.Email == brokerEmail &&
                r.RegistrationType == RegistrationType.Makler &&
                r.ParentRegistrationId == null)
            .OrderByDescending(r => r.RegistrationDateUtc)
            .ToListAsync();
    }
    ```

    Filter criteria:
    - `r.Email == brokerEmail` — SQL Server collation handles case-insensitivity server-side (per RESEARCH.md pitfall 3)
    - `r.RegistrationType == RegistrationType.Makler` — broker registrations only
    - `r.ParentRegistrationId == null` — top-level registrations only (guests are loaded via GuestRegistrations nav prop)
    - Do NOT filter by `!r.IsCancelled` — cancelled registrations must be shown with "Storniert" badge per locked decision
    - GuestRegistrations are included without IsCancelled filter — all guest states visible inline

    No new using statements needed (RegistrationType and DbContext already imported).
  </action>
